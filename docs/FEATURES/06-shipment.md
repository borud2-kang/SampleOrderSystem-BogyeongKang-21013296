# 기능 명세: 출고 처리

관련 PRD: [PRD.md § 6.7](../PRD.md#67-출고-처리)

## 개요

재고가 충분해져 출고 대기 중인(`CONFIRMED`) 주문에 대해 실제 출고를 처리하는 기능. 주문 생명주기의 마지막
단계다.

## 하위 기능

### 출고 처리

1. `CONFIRMED` 상태인 주문 목록을 표시한다.
2. 담당자가 목록에서 특정 주문을 선택한다.
3. 선택한 주문의 상태를 `RELEASE`로 전환한다.

`CONFIRMED` 상태는 이미 재고 확보(또는 생산 완료)가 끝난 상태를 의미하므로, 출고 처리 시점에 재고를 다시
확인하거나 차감할 필요는 없다 — 재고 차감은 [주문 승인](03-approval.md#2-주문-승인) 또는
[생산 완료](05-production-line.md#생산-완료-처리) 시점에 이미 반영되어 있어야 한다.

## 엣지 케이스 / 검증 규칙

- `CONFIRMED`가 아닌 주문(예: 이미 `RELEASE`된 주문, 아직 `PRODUCING`/`RESERVED`인 주문)에 대한 출고 시도는
  거부한다.
- 존재하지 않는 주문번호 선택 시 에러 처리.
- 출고 처리 시각을 기록해 두면 모니터링/이력 조회에 활용할 수 있다 (필수는 아니나 권장).
