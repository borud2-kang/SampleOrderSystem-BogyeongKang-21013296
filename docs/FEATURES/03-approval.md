# 기능 명세: 주문 승인/거절

관련 PRD: [PRD.md § 6.4](../PRD.md#64-주문-승인거절)

## 개요

생산 라인 담당자가 접수된(`RESERVED`) 주문을 확인하고 승인 또는 거절을 결정한다. 이 기능이 이 시스템에서
가장 핵심적인 분기 로직(재고 상태에 따른 자동 상태 전환)을 담당한다.

## 하위 기능

### 1. 접수된 주문 목록

- `RESERVED` 상태인 주문만 필터링해 표시한다.
- 표시 항목: 주문번호, 고객명, 시료(명), 주문 수량, 상태.

### 2. 주문 승인

담당자가 목록에서 특정 주문을 선택해 승인하면:

1. 대상 시료의 **현재 재고**를 조회한다.
2. **재고가 주문 수량 이상인 경우**(재고 충분):
   - 재고에서 주문 수량만큼 차감한다.
   - 주문 상태를 즉시 `CONFIRMED`로 전환한다.
3. **재고가 주문 수량 미만인 경우**(재고 부족):
   - 부족분 = 주문 수량 − 현재 재고.
   - 해당 주문을 생산 라인의 대기열(생산 큐)에 자동으로 등록한다. 생산량/시간 계산 로직은
     [`05-production-line.md`](05-production-line.md) 참고.
   - 주문 상태를 `PRODUCING`으로 전환한다.
   - 이 시점에 재고 전량을 미리 차감할지, 생산 완료 시 차감할지는 구현 선택 사항이지만, **이중 차감이
     발생하지 않도록** 일관된 규칙을 정하고 테스트로 고정해야 한다 (권장: 생산 완료 후 재고 갱신 및 즉시
     소비 처리).

### 3. 주문 거절

- 담당자가 목록에서 특정 주문을 선택해 거절하면 주문 상태를 즉시 `REJECTED`로 전환한다.
- `REJECTED`는 정상 흐름 외 상태이며, 이후 모든 모니터링/통계 집계에서 제외된다.
- 재고나 생산 큐에 영향을 주지 않는다 (애초에 승인 전이므로 아직 반영된 바가 없다).

## 상태 분기 요약

```
RESERVED --[승인, 재고 >= 주문수량]--> CONFIRMED
RESERVED --[승인, 재고 <  주문수량]--> PRODUCING
RESERVED --[거절]--------------------> REJECTED
```

## 엣지 케이스 / 검증 규칙 (중점 테스트 대상)

- 재고와 주문 수량이 **정확히 같은 경우**: "충분"으로 간주해 `CONFIRMED`로 처리한다 (부족분 0).
- 재고가 0인 경우: 부족분 = 주문 수량 전체, `PRODUCING`으로 전환.
- 이미 `RESERVED`가 아닌 주문(이미 승인/거절/처리된 주문)에 대한 재승인·재거절 시도는 거부한다.
- 존재하지 않는 주문번호 선택 시 에러 처리.
- 동시에 여러 `RESERVED` 주문이 같은 시료를 두고 경쟁하는 경우, 승인 처리 순서(담당자가 목록에서 선택한
  순서)대로 재고를 소비한다 — 별도의 우선순위 로직은 없다.
