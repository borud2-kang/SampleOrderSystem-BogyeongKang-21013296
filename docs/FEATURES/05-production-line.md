# 기능 명세: 생산 라인

관련 PRD: [PRD.md § 6.6](../PRD.md#66-생산-라인)

## 개요

공장에서 시료를 생산하는 설비 흐름을 관리한다. 하나의 생산 라인은 시료를 하나씩 생산하며, **주문이 들어온
시료에 대해서만** 생산한다 (재고를 미리 쌓아두기 위한 예비 생산은 하지 않는다).

## 생산 트리거

[주문 승인](03-approval.md)에서 재고가 부족하다고 판단된 주문만 생산 큐에 등록된다. 재고가 이미 충분한
주문은 생산 라인을 거치지 않고 바로 `CONFIRMED`가 된다.

## 생산량/시간 계산

| 항목 | 산식 |
| --- | --- |
| 부족분 | 주문 수량 − 현재 재고 |
| 실 생산량 | `ceil(부족분 / 수율)` |
| 총 생산 시간 | 평균 생산시간 × 실 생산량 |

- 수율이 1 미만이므로 불량품 발생을 감안해 부족분보다 더 많은 수량을 생산해야 한다.
- 예: 부족분 170, 수율 0.92 → 실 생산량 = `ceil(170 / 0.92)` = 185 (PDF 예시는 어림값이므로 구현 시
  정확한 올림 계산을 기준으로 한다).
- **재고 반영 정책 (수업 규칙 — 명시적 약속)**: 수율의 원래 정의는 "정상품 수 / 총 생산량"이지만, 이
  프로젝트는 불량품/정상품을 실제로 나누어 추적하지 않기로 한다. 수율은 오직 부족분을 채우기 위해
  "얼마나 더 많이 생산 지시를 내려야 하는가"(`ceil(부족분/수율)`)를 정하는 데만 쓰고, 일단 생산이 끝나면
  **생산된 실 생산량(`actual_qty`) 전체를 정상품으로 간주해 재고에 그대로 반영**한다(수율을 다시 곱해
  "실제 정상품 수"를 역산하지 않는다). 개념적으로 "수율만큼만 정상품이 나온다"는 정의와는 다소 어긋나
  보일 수 있으나, 이 프로젝트가 채택한 명시적 단순화 규칙이다. 참고로 `floor(ceil(부족분/수율) × 수율)`은
  부동소수점 오차를 제외하면 수학적으로
  항상 부족분과 정확히 같아지므로(올림으로 생긴 여유가 곱셈+내림에서 정확히 상쇄됨), 수율을 재적용해
  "정상품 수"를 계산하는 방식으로는 애초에 잉여 재고가 발생할 수 없다. 실제로 잉여(`actual_qty - 부족분`)를
  재고에 남기려면 실 생산량 전체를 그대로 반영하는 이 방식이 필요하다 — 즉 "생산 지시량만큼은 전부
  정상품으로 재고에 들어온다"고 가정하고, 수율은 "얼마나 더 많이 생산 지시를 내려야 하는가"를 정하는
  용도로만 쓴다. 이 잉여는 그대로 재고에 남아 이후 다른 주문에 재사용될 수 있다.

## 스케줄링

- 생산 큐는 **FIFO(선입선출)** 로 동작한다. 우선순위 재정렬, 새치기 없음.
- 한 번에 하나의 주문(시료)만 처리한다 (단일 생산 라인 가정, [PRD § 3](../PRD.md#3-범위) 참고).

## 생산 완료 처리

- 생산이 완료되면:
  1. 해당 시료의 재고를 갱신한다(부족분만큼 증가, 또는 정책에 따라 즉시 소비 처리).
  2. 주문 상태를 `PRODUCING → CONFIRMED`로 전환한다.
  3. 큐에서 다음 대기 주문을 꺼내 생산을 시작한다.

## 하위 기능 (조회)

### 1. 생산 현황 표기

- 현재 생산 중인 시료에 대한 정보를 표시한다.
- 표기할 정보 수준은 자율적으로 결정한다 (예: 주문 정보, 실 생산량, 진행률, 예상 완료 시각 등).

### 2. 대기 주문 확인

- 생산 큐에서 대기 중인 주문 목록을 FIFO 순서대로 표시한다.
- 각 대기 주문에 대해 부족분/실생산량/예상 완료 시각 등을 함께 표시할 수 있다 (PDF 21쪽 예시 참고, 필수
  아님).

## 엣지 케이스 / 검증 규칙 (중점 테스트 대상)

- 수율이 정확히 1인 경우: 실 생산량 == 부족분.
- 부족분을 수율로 나눈 값이 정수인 경우와 아닌 경우 모두 `ceil` 이 올바르게 적용되는지 검증 (예:
  180/0.9=200 정확히 나눠떨어지는 경우도 반드시 올림 함수로 처리해 부동소수점 오차로 200.000001 같은 값이
  201로 올림되지 않도록 주의).
- 큐가 비어 있는 상태에서 생산 라인 조회 시 "대기 없음" 등 정상 처리.
- 하나의 생산이 끝나기 전에 새 주문이 승인되어 큐에 추가되는 경우, 기존 진행 중인 생산에는 영향을 주지 않고
  큐 맨 뒤에 추가된다.
