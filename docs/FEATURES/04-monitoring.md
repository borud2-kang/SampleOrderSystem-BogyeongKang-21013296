# 기능 명세: 모니터링

관련 PRD: [PRD.md § 6.5](../PRD.md#65-모니터링)

## 개요

담당자가 현재 시스템의 상태(주문 현황, 재고 현황)를 한눈에 파악할 수 있도록 하는 조회 전용 기능. 데이터를
변경하지 않는다.

## 하위 기능

### 1. 주문량 확인

- 상태별(`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`) 주문 건수를 집계해 표시한다.
- **`REJECTED`는 유효한 주문이 아니므로 집계에서 제외**한다 — 이 규칙은 통계 화면 전체에 일관되게
  적용되어야 한다 (총 주문 수를 표시할 때도 REJECTED는 포함하지 않는다).

### 2. 재고량 확인

- 각 시료별 현재 재고 수량을 표시한다.
- 주문 대비 재고 수량에 따라 파생 상태를 함께 표기한다:

| 상태 | 판정 기준(예시) |
| --- | --- |
| 여유 | 미결(RESERVED/PRODUCING 등 아직 출고되지 않은) 주문 수요 대비 재고가 충분 |
| 부족 | 재고가 0은 아니지만 미결 주문 수요에 못 미침 |
| 고갈 | 재고 수량이 정확히 0 |

정확한 임계값 산식(예: 잔여율 %)은 명세에 구체적으로 주어지지 않았으므로, 구현 시 다음을 권장 기본값으로
삼는다: 재고 == 0 → 고갈, 0 < 재고 < 해당 시료에 대한 미결 주문 총수량 → 부족, 그 외 → 여유. 필요하면
사용자와 상의해 조정한다.

## 엣지 케이스 / 검증 규칙

- 등록된 시료가 없는 경우, 주문이 하나도 없는 경우 각각 빈 상태를 명확히 표시한다 (에러로 처리하지 않음).
- 재고량 확인 화면의 파생 상태는 매 조회 시점에 다시 계산한다 (캐시된 값 사용 금지 — 재고/주문은 계속
  변경되는 값이므로).
