# 기능 명세: 시료 관리

관련 PRD: [PRD.md § 6.2](../PRD.md#62-시료-관리)

## 개요

시료(Sample)는 이 시스템의 가장 기본이 되는 단위다. 각 시료는 고유한 이름과 속성을 가지며, **시스템에
등록된 시료만 주문 가능**하다.

## 데이터 모델

| 속성 | 타입 | 설명 |
| --- | --- | --- |
| 시료 ID | string | 고유 식별자 (예: `S-001`) |
| 이름 | string | 시료명 (예: `실리콘 웨이퍼-8인치`) |
| 평균 생산시간 | number | 1개 생산에 소요되는 평균 시간(분 등 단위는 구현자 결정) |
| 수율 | number (0~1) | 정상적인 시료 수 / 총 생산 시료 수. 예: 100개 생산 중 정상 90개 → 0.9 |
| 현재 재고 | number | 즉시 출고 가능한 수량. 등록 시점에는 0 또는 초기값으로 시작하고, 이후 생산 완료/출고에 따라
  증감 |

> **구현상 유의(수업 규칙)**: 수율의 정의는 위 표와 같지만, 이 프로젝트는 불량품/정상품을 실제로 나눠
> 추적하지 않는다 — 수율은 생산 지시량을 정하는 데만 쓰이고, 생산된 수량 전체를 정상품으로 간주해
> 재고에 반영한다. 상세는 [`05-production-line.md`](05-production-line.md) "재고 반영 정책" 절 참고.

## 하위 기능

### 1. 시료 등록

- 새로운 시료를 시스템에 추가한다.
- 입력값: 이름, 평균 생산시간, 수율.
- 시료 ID는 사용자가 직접 입력하지 않고 시스템이 자동으로 채번한다 (예: `S-001`, `S-002`, ...
  [`docs/design/phase0.md`](../design/phase0.md#persistence-설계) 참고). 이 방식으로 시료 ID는 항상
  고유함이 보장되며, 저장소 계층(`SampleRepository.create`)도 방어적으로 중복 여부를 검증한다.
- 수율은 0을 초과하고 1 이하인 값이어야 한다 (0이면 `ceil(부족분 / 수율)` 계산 시 0으로 나누는 문제가
  발생하므로 반드시 검증).

### 2. 시료 조회

- 등록된 모든 시료 목록을 표시한다.
- 각 시료의 **현재 재고 수량도 함께 표시**한다.
- 목록이 많을 경우 페이지네이션 등은 구현자 재량 (예시 UI의 "다음페이지" 참고, 필수는 아님).

### 3. 시료 검색

- 이름 등 속성으로 특정 시료를 검색한다.
- 검색 방식(정확히 일치/부분 일치)은 구현자 재량.

## 엣지 케이스 / 검증 규칙

- 이 화면(시료 관리)에는 ID로 직접 조회하는 메뉴가 없다 (검색은 이름 기준). 존재하지 않는 시료 ID를
  다루는 시나리오는 주문 생성 시 발생하며, 해당 검증은
  [`docs/FEATURES/02-order.md`](02-order.md)에서 다룬다.
- 시료 등록 시 필수값 누락, 숫자 형식 오류(생산시간/수율에 문자열 입력 등) 처리.
- 이름 중복은 허용되는지(시료 ID만 고유해야 하는지) 여부는 명세에 명시되어 있지 않음 — 구현 시 시료 ID
  고유성만 강제하고, 이름 중복은 허용하는 쪽을 기본값으로 한다.
