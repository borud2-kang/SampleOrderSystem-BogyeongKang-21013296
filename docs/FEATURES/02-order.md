# 기능 명세: 시료 주문 (예약)

관련 PRD: [PRD.md § 6.3](../PRD.md#63-시료-주문-예약)

## 개요

고객이 시료를 요청하면 주문 담당자가 시스템에 주문을 생성한다. 이 기능은 주문 생명주기의 시작점으로,
생성된 주문은 항상 `RESERVED` 상태에서 출발한다.

## 데이터 모델

| 속성 | 타입 | 설명 |
| --- | --- | --- |
| 주문번호 | string | 고유 식별자 (예: `ORD-20260416-0043`) |
| 시료 ID | string | 주문 대상 시료 (등록된 시료여야 함) |
| 고객명 | string | 주문한 고객 |
| 주문 수량 | number | 요청 수량 (양의 정수) |
| 상태 | enum | 생성 직후 `RESERVED` (전체 상태값은 [상태 전이](../PRD.md#53-주문-상태-order-status) 참고) |

## 하위 기능

### 시료 예약 (주문 생성)

입력값:

1. 시료 ID
2. 고객명
3. 주문 수량

처리 흐름:

1. 시료 ID가 등록된 시료인지 검증한다. 미등록 시료는 주문할 수 없다.
2. 주문 수량이 1 이상의 정수인지 검증한다.
3. 신규 주문번호를 발급하고, 상태를 `RESERVED`로 설정해 저장한다.
4. 결과(주문번호, 상태)를 확인 화면에 표시한다.

이 시점에서는 **재고 확인을 하지 않는다** — 재고 확인 및 상태 분기는 "주문 승인/거절" 기능([`03-approval.md`](03-approval.md))에서 처리한다.

## 엣지 케이스 / 검증 규칙

- 존재하지 않는 시료 ID 입력 시 주문 생성을 거부하고 재입력을 요구한다.
- 수량 0 또는 음수, 숫자가 아닌 입력에 대한 검증.
- 동일 고객이 동일 시료를 중복 주문하는 것은 제한하지 않는다 (별개 주문으로 각각 처리).
